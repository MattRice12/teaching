Overview

Functions are essential to concise, readable, and purposeful code. They are like real-world factories in that they take raw materials, process these materials, and yield a product for us to use.

Thus a function has 3 parts:

  1. The arguments given to the function,
  2. The processing part of the function, and
  3. The return value of the function.

Function definition

The function definition is the factory before we give it any raw materials. Let's take a look at a function definition:

function (arg1, arg2) {
let result = arg1 + arg2;
return result;
};

`arg1` and `arg2` are the arguments of the function. Technically, since this is a function definition, `arg1` and `arg2` are called parameters. As described below, when we call/execute the function, the values we put in are the arguments.

Everything inside the curly braces is the body of the function. Here, we are adding the values arg1 and arg2 together and then returning that result.

The `return` keyword stops the function and outputs the proceeding value wherever the function is called.

Function Call

To make the factory work, we need to give it some raw material as inputs. We give the function these raw materials when we `call` it.

In addition to calling the function, I'm also:

  1. Making it more concise,
  2. Giving it a name, sum,
  3. Assigning the return value of sum() to the variable, `total`.

function sum(arg1, arg2) {
return arg1 + arg2;
};

let total = sum(2, 3);

`sum(2, 3)` is our function call. The integers `2` and `3` are the arguments being passed in. The function adds the two values together and returns `5`. We then assign the returned value to the variable `total`.

Return

The return value is a particularly important part of the function, such that it deserves its own section.

As mentioned above, the return does two things:

  1. The value after the return is the value of the function, and
  2. Return terminates the function when it is reached.

We've seen the first part above with `sum(2, 3)`. Since the function returns `2 + 3`, the value of the function is 5. Thus when we assign `sum(2, 3)` to `total`, `total` has the value 5. (*Note, this applies only to pure functions.)

If the `return` is left off the function, the functions value is `undefined`.

Since the function terminates when it reaches the return, any code written below the return is ignored. Code written below a return will typically only be seen when a function has a conditional with a return. Take the following for example:

function size(num) {
if (num > 5) {
return "large";
};
return "small";
};

In the above example, the function terminates and returns "large" if the value of `num` is greater than 5. If the value of `num` is less than or equal to 5, it skips the conditional body and returns "small".
